# RealTrack OS — Architecture Program

RealTrack is evolving from an application into an **AI Operating System for
real-estate acquisition companies**. Per the product constitution, every module
is designed before it is built. This directory is the single source of truth
for that design work.

## Rules of the program

1. No module is implemented before its design section is approved.
2. Every module design must specify: Purpose · Users · Business Rules ·
   Database Tables · Events · API Endpoints · AI Usage · Permissions ·
   Edge Cases · Success Metrics · Acceptance Criteria.
3. The existing production app keeps shipping; OS work lands incrementally
   behind the same event-driven backbone (no big-bang rewrite).
