feat(arv): Zillow = data only (Zestimate + comps); Gemini computes the ARV/MAO

Clear separation of duties:
  • Zillow API → fetches DATA only: Zestimate + property facts + comparable sales.
  • Gemini → qualifies the lead AND computes the ARV from those comparables
    (median $/sqft × subject sqft), with arv_reasoning showing the math.

/api/analyze: ensures market data (re-pulls cache-backed /api/zillow when the
lead lacks a Zestimate OR comparables — covers inbound-API leads too), injects a
MARKET DATA block (subject facts + up to 12 comps) into the Gemini prompt, and
reads back estimated_arv + arv_reasoning (new QualJSON/responseSchema fields).
Persists metadata.arv = Gemini's ARV, metadata.zestimate, metadata.comparables,
metadata.arv_reasoning. If Zillow has no Zestimate, the verdict now falls back to
Gemini's comp-based ARV as the market value — fixes leads stuck on "no usable
price/market value". Removed the old /api/leads/arv dependency from the analyze
auto-correction path.

UI (AcquisitionsPanel): the MAO card no longer says "Zillow ARV". It now shows
"Zestimate (Zillow)" as a market-value reference and "AI-Estimated ARV"
(with the AI's derivation as subtext) — the ARV that drives the MAO. Offer PDF
labels updated to match. Lead page passes zestimate + arv (Gemini) separately
instead of falling ARV back to the Zestimate.
